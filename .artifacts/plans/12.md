# Plan — Issue #12: Add ~30 community palettes

Domain: `palette-community`
Branch: `feat/palette-community`
Target: ship ~30 community palette atoms validated against `tools/schemas/palette.ts`.

## Approach

Each palette = its own directory `palettes/<slug>/1.0.0/atom.yaml`. Zero cross-file
coupling: coders working on different palettes can never collide.

Partition into 3 groups, hand-authored from canonical upstream sources.

## Authoring discipline

Each atom MUST:
1. `kind: palette`, `id` matching folder slug, `version: 1.0.0`
2. `swatches[]` with stable slug IDs and real hex values from canonical source
3. Logical swatch grouping with comments (bg / fg / accents)
4. Both `modes.light.roles` and `modes.dark.roles` populated — for single-mode
   palettes (Dracula, Tokyo Night Storm etc.), the secondary mode is a thoughtful
   inversion swapping bg/fg while preserving accents
5. `provenance.source` URL pointing at the canonical upstream
6. `provenance.license` SPDX identifier
7. `provenance.attribution` real author/team
8. `provenance.importedDate: 2026-05-17`
9. `tags[]` meaningful (e.g., `[dev-theme, dark, classic]`)
