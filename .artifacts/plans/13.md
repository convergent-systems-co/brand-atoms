# Plan — Issue #13: Bulk-import ~100 Google Fonts via METADATA.pb

## Goal

Build a deterministic, idempotent import script at `tools/imports/google-fonts.ts` that:

1. Accepts a list of font slugs (with optional license-directory hint).
2. Fetches each slug's canonical `METADATA.pb` from `https://raw.githubusercontent.com/google/fonts/main/{ofl|apache|ufl}/<slug>/METADATA.pb`.
3. Parses the protobuf-text format (tiny hand-rolled state-machine parser — no protobuf.js dep).
4. Emits a schema-valid `fonts/<slug>/<version>/atom.yaml`.
5. Is fully deterministic: re-running with the same slug list produces zero diff (modulo upstream changes).

Then run the importer to ship ~100 atoms across all six classifications.

## Schema mapping (from METADATA.pb → Font atom)

| METADATA field        | Atom path                                |
|-----------------------|------------------------------------------|
| `name`                | `name`, `family`, `source.family`        |
| Folder slug           | `id` (lowercased, normalized; e.g. `Open Sans` → `opensans`) |
| `designer`            | `provenance.attribution` (designer line) |
| `category`            | `classification` (mapped: SANS_SERIF→sans-serif, SERIF→serif, MONOSPACE→monospace, DISPLAY→display, HANDWRITING→handwriting) |
| `license` + dir       | `provenance.license` (OFL → `OFL-1.1`, APACHE2/dir=apache → `Apache-2.0`, UFL → `UFL-1.0`) |
| `fonts {}` blocks     | `availableStyles[]` (each weight/style)  |
| `fonts[0].copyright`  | folded into `provenance.attribution`     |
| METADATA URL          | `provenance.source`                      |
| `date_added`          | not used; we use today's ISO date for `importedDate` |
| `axes {}` blocks      | logged in `description` if variable      |

### Family → Google CSS URL

Build a `https://fonts.googleapis.com/css2?family=<Family+Name>:ital,wght@...&display=swap` URL using all unique `(italic, weight)` pairs from the METADATA `fonts {}` blocks, sorted as `0,w1;0,w2;...;1,w1;1,w2;...`.

If a font has only normal styles, use the simpler `:wght@...` axis form. Variable fonts (with `wght` axis) get a range form `:wght@<min>..<max>`.

### Fallback stacks by classification

- `sans-serif`: `-apple-system, BlinkMacSystemFont, Segoe UI, Roboto, Helvetica Neue, Arial, sans-serif`
- `serif`: `Georgia, Cambria, Times New Roman, Times, serif`
- `slab-serif`: `Rockwell, Courier Bold, Courier, Georgia, Times, serif`
- `monospace`: `Menlo, Monaco, Consolas, Liberation Mono, Courier New, monospace`
- `display`: `Georgia, Cambria, serif` (display fonts vary; conservative)
- `handwriting`: `Brush Script MT, Lucida Handwriting, cursive`

### Slab-serif detection

Google's enum has no SLAB_SERIF. We detect slab fonts by checking if the slug or name contains "slab" (e.g., `robotoslab`, `arvo`, `bitter`, `zillaslab`, `merriweather`). We special-case a curated set in the script and tag them with `slab` + classification `slab-serif`.

## Risk register

- **Variable-font weight ranges**: METADATA `fonts {}` blocks list discrete static-style entries with weight numbers; variable axis info is in `axes {}` blocks. The script enumerates the discrete `fonts {}` entries for `availableStyles[]` (this is the schema-correct interpretation: `availableStyles` is a list, not a range). If a `wght` axis exists, the CSS URL uses the `wght@min..max` range form for accurate Google delivery.
- **Multiple weights with same (weight,style)**: dedup before emit.
- **Quirky filenames vs slugs**: METADATA name normalization → slug must match Google's directory convention (lowercase, no spaces). We pre-normalize the slug list and use `slugify(name)` for the emitted `id`.
- **Apache vs OFL mismatches**: cross-check both `license` field in METADATA and the directory. Directory wins for SPDX mapping (e.g., apache/ → Apache-2.0).
- **Network flakiness**: ≤3 retries with exponential backoff per fetch.
